2. Naming Conventions
	•	Files: snake_case → lexer.cpp, ast_node.hpp.
	•	Namespaces: camelCase or PascalCase (mylang::parser, mylang::runtime).
	•	Classes / Structs: PascalCase → Lexer, Parser, AstNode, Interpreter.
	•	Methods: camelCase → nextToken(), parseExpression().
	•	Variables: snake_case → current_token, line_number.
	•	Constants / Enums: UPPER_CASE → TOKEN_IF, AST_NODE_TYPE.
	•	Private members: suffix _ → token_stream_, line_.
	•	Global symbols: prefix with project namespace → mylang::init().

⸻

3. Lexer / Tokenizer Conventions
	•	Keep Token immutable after creation.
	•	Token class fields: type, lexeme, line, column.
	•	Define token types as enum class TokenType.
	•	Provide toString() for debugging.
	•	Avoid mixing lexing and parsing logic—lexer only produces tokens.

⸻

4. Parser Conventions
	•	Choose recursive descent or table-driven (LL(1), LR) and stick with it.
	•	Each grammar rule = one function (parseExpression(), parseStatement()).
	•	Always consume tokens with a clear helper: match(TokenType), expect(TokenType).
	•	Keep AST node creation in factory functions.

⸻

5. AST (Abstract Syntax Tree) Conventions
	•	Use a base class AstNode with subclasses (BinaryExpr, IfStmt, FuncDecl).
	•	Each node has:
	•	type tag (enum AstNodeType).
	•	children (vector of nodes or owned pointers).
	•	Provide accept(Visitor&) for traversals (Visitor Pattern).
	•	Keep AST printing (dump() or prettyPrint()) separate from semantic analysis.

⸻

6. Semantic Analysis
	•	Store symbol tables in a Context object.
	•	Separate type checking, name resolution, and scope management.
	•	Fail fast: throw structured errors (SemanticError).

⸻

7. Interpreter / Compiler Backend
	•	Interpreter:
	•	Walk the AST with a Visitor.
	•	Use Value type for runtime values (like LLVM’s APInt / APFloat).
	•	Compiler backend:
	•	Convert AST → IR (Intermediate Representation).
	•	Keep IR printing/debugging separate from optimization.
	•	Runtime:
	•	Provide standard library (print, len, etc.) as separate module.

⸻

8. Error Handling
	•	Lexing/Parsing Errors:
	•	Show line/column + snippet.
	•	Use structured error type (ParseError).
	•	Runtime Errors:
	•	Include stack trace if possible.
	•	Always provide error(message, line, column) utility.

⸻

9. Testing Conventions
	•	Unit tests for lexer (input → tokens).
	•	Parser tests (input → AST shape).
	•	Semantic tests (input → type errors or symbol table).
	•	Execution tests (input code → expected output).
	•	Use golden tests (store expected output in files).

⸻

10. Style & Documentation
	•	Stick to Google C++ Style Guide or LLVM Coding Standards.
	•	Each class/function gets a docstring explaining role.
	•	Add EBNF grammar in /docs/grammar.ebnf.
	•	Add design docs: architecture, AST design, error handling strategy.

⸻

11. Performance & Memory
	•	Use std::unique_ptr for AST ownership.
	•	Avoid raw pointers unless interfacing with C APIs.
	•	Use reserve() when token/AST size is predictable.
	•	Provide profiling hooks (--time-tokens, --time-parse).

⸻

12. Version Control & CI
	•	Git:
	•	main branch stable, dev for new features.
	•	Use clang-format to enforce style.
	•	CI:
	•	Build on Linux + macOS.
	•	Run all tests automatically.

Indentation logic : 

    FUNCTION generate_indents(tokens):
        indents = [0]
        stream = []

        WHILE tokens IS NOT EMPTY:
            token = tokens[0]
            REMOVE FIRST ELEMENT FROM tokens

            IF token.type == 'NL':
                IF tokens IS EMPTY:
                    APPEND token TO stream
                    CALL dedent(0, indents, stream)
                    BREAK LOOP

                ELSE IF tokens[0].type == 'WS':
                    indent = LENGTH OF tokens[0].value

                    IF indent > LAST ELEMENT OF indents:
                        APPEND indent TO indents
                        APPEND ('Indent', indent) TO stream

                    ELSE IF indent == LAST ELEMENT OF indents:
                        APPEND token TO stream

                    ELSE:
                        APPEND token TO stream
                        CALL dedent(indent, indents, stream)

                    REMOVE FIRST ELEMENT FROM tokens  // consume the WS token

                ELSE:
                    APPEND token TO stream
                    CALL dedent(0, indents, stream)

            ELSE:
                APPEND token TO stream

        ASSERT LENGTH OF indents == 1
        RETURN stream



    FUNCTION dedent(indent, indents, stream):
        WHILE indent < LAST ELEMENT OF indents:
            REMOVE LAST ELEMENT FROM indents
            APPEND ('Dedent', LAST ELEMENT OF indents) TO stream

        ASSERT indent == LAST ELEMENT OF indents
        RETURN
